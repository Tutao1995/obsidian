# STACK CHECK LIST

技术栈检测与文档索引。编码和审查前，按检测条件自动加载对应文档。

## 默认技术栈

无特殊说明时，项目默认技术栈为 **Java 8** + **Vue 2**。

## 使用方式

1. **检测到技术栈** → 阅读对应的最佳实践 + 代码规则文档
2. **多技术栈项目** → 阅读所有匹配的文档
3. **无匹配文档** → 跳过此步骤，继续实施
4. 阅读后声明："我已阅读：[文档名1]、[文档名2]、..."

## 索引表

| 技术栈 | 检测条件 | 最佳实践 & 代码规则 | 测试策略 | Risks/Rollout |
|--------|---------|-------------------|---------|---------------|
| **Java** | 编写/审查 Java 代码 | [BACKEND-JAVA.md](./BACKEND-JAVA.md) | 见文档内"测试策略"章节 — **必须编写单测** | 见文档内"风险评估"和"发布策略"章节 |
| **Java + Seeyon** | 编写/审查 Seeyon 后端代码 | [BACKEND-JAVA-SEEYON.md](./BACKEND-JAVA-SEEYON.md)（需同时阅读 BACKEND-JAVA.md） | 同 Java | 同 Java |
| **Vue** | 编写/审查 Vue 代码 | [FRONTEND-VUE.md](./FRONTEND-VUE.md) | 见文档内"测试策略"章节 — **可选，询问用户** | 见文档内"风险评估"和"发布策略"章节 |
| **前端 API** | 从 Swagger 生成前端 API 文件 | [FRONTEND-API.md](./FRONTEND-API.md) | 不适用 | 不适用 |
| **Raw** | 用户指定 | 阅读用户指定的文档 | 按用户要求 | 按用户要求 |

## 扩展方式

新增技术栈规则时，在 `.best-practices/` 目录下创建对应 `.md` 文件，并在上表中添加一行索引。文件需包含：核心原则、通用规范、测试策略、风险评估触发条件、发布策略触发条件（如适用）。
