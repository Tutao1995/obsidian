## UI Spec 文档模板

### Why

Step 5 生成的 UI Spec 文档的标准结构，确保每次输出格式一致。

### What

完整的 Markdown 文档骨架，包含文件清单、功能概述、整体视图、各区域详情、状态变体。

### How

```markdown
# {项目名称} — UI 规格说明书

## 文件清单

| 文件 | 说明 |
|------|------|
| `ui_spec_{项目名}.md` | 本文档 |
| `{group}_preview.html` | 完整 HTML 预览（浏览器打开验证还原效果） |
| `{group}_styles.md` | 精确样式基线数据 |
| `{group}_tree.txt` | 结构化节点树（Framelink 模式） |
| `截图文件.png` | Figma 原图截图 |

## 1. 功能概述

简要描述功能用途和交互目标。

## 2. 整体视图（默认状态）

![截图](basic_default.png)

```ascii 结构图```

- 布局描述（按 SKILL.md 5.1 规范）
- 精确样式规格（尺寸、背景色、间距）
- HTML/CSS 代码片段（从 {group}_preview.html 提取的整体骨架）

## 2.x 精确样式基线（来自该分组的 figma-extract.js 输出）

嵌入该分组代表 frame 的样式数据：容器尺寸、色板、字体、布局、圆角、阴影等。

## 3-N. 各区域详情（按 UI 区域拆分）

每个区域包含：
1. 高保真截图（裁剪或标注区域）
2. ASCII 结构图（展示布局和组件层次）
3. 布局描述（按 SKILL.md 5.1 规范，精确描述对齐/间距/列结构/嵌套关系）
4. HTML/CSS 代码片段（从 {group}_preview.html 提取的该区域代码）
5. 精确样式规格（从该分组的样式提取结果匹配，直接嵌入本节）

## M. 状态变体（必须覆盖所有状态 frame）

按业务场景分组，每个状态包含：
1. 截图 + ASCII 结构图
2. 与默认状态的差异说明（哪些区域变了、变成什么样）
3. 触发条件（什么操作/数据导致这个状态）
4. 差异样式标注（仅列出与同组基线不同的样式值，如新增颜色、不同尺寸）

常见状态维度：
- 空态 / 加载中 / 默认 / 部分成功 / 全部成功 / 部分失败 / 全部失败
- 可编辑 / 只读 / 禁用
- 附件类型差异（文件/按钮/文本）
```

## 布局描述示例

### Why

SKILL.md 5.1 中布局描述的参考标准，展示逐子区域精确描述的写法。

### What

配置弹窗的完整布局描述示例，覆盖弹窗整体、头部、提示栏、分组卡片、字段区域、底部等。

### How

```markdown
#### 布局描述

弹窗整体
- 弹窗居中于视口，宽度约占屏幕 60-70%，高度自适应内容（内容多时可滚动）。
- 内部内容区左右有统一的 padding（约 24-32px）。

头部
- 图标与标题垂直居中、水平左对齐；关闭按钮垂直居中、水平右对齐，与标题同一行。

警告提示栏
- 水平撑满弹窗内容区（左右与内容区齐平），图标与文字垂直居中。

分组卡片
- 各卡片纵向堆叠，之间有约 16-24px 的间距。
- 每个卡片水平撑满内容区。

卡片标题行
- 绿色竖线紧贴标题文字左侧，与文字垂直居中对齐。
- 折叠箭头固定在卡片右侧，与标题文字垂直居中。

字段区域内部布局（关键）
采用三列网格结构：[权限标签列] [左字段列] [右字段列]
- 权限标签列：固定宽度（约 80px），位于最左侧。标签（"可编辑"/"不可编辑"）在其所属字段组的第一行垂直居中显示，后续行该列留空。
- 左字段列 / 右字段列：两列等宽，中间有约 16px 的间隔（gutter）。当某行只有一个字段时，仅占左列，右列留空。

单个字段内部对齐
- 绿色竖线高度与字段行等高，紧贴字段内容左侧（作为左边框装饰），与字段之间有约 8px 间距。
- 字段名称左对齐，字段类型标签（"文本"/"数字"/"日期控件"）右对齐，两者在同一行内垂直居中。
- 字段行之间无分割线，行间距约 8-12px。

各行垂直对齐关系
- 同一行的左字段与右字段顶部对齐（baseline 对齐）。
- 左列的绿色竖线与右列的绿色竖线在水平方向上分别与各自列的左边缘对齐，不跨列连续。

底部关闭按钮
- 位于弹窗底部区域，水平右对齐，与内容区右边缘齐平，距底部有约 16-24px 的 padding。
```

## Framelink 节点树输出示例

### Why

SKILL.md 4.3 中 Framelink 模式节点树的格式参考。

### What

配置弹窗（frame 45098:13274）的实际输出示例，展示缩进文本格式和每个节点的属性标注方式。

### How

```

  │       ├─ Frame 1912056616 [FRAME] 712×24 row/center/gap:8 fill×hug
  │       │   ├─ Frame 1000001524 [FRAME] 24×24 bg:linear-gradient(180deg,...)
  │       │   └─ 查看抽取配置 [TEXT] "查看抽取配置" 16px/500/#000000
  │       └─ tool [FRAME] 16×16
  │           └─ 关闭 [FRAME] 16×16 stroke:#666

  │   │   └─ cap-icon-guanbi [FRAME] 14×14
  │   ├─ Frame 2043683591 [FRAME] 752×222 (主表分组)
  │   │   ├─ Frame 2043683587 [FRAME] 752×80 bg:#FAFAFA r:20,20,0,0 (标题壳)
  │   │   │   └─ ...
  │   │   └─ Frame 2043683598 [FRAME] 752×182 bg:#FFFFFF border:#F0F0F0/1px r:20 (内容卡片)
  │   │       └─ ...
  │   └─ ...
  └─ footer [FRAME] 800×60 row/flex-end/center/gap:8/pad:14,24 shadow:inset-top
```

**每个节点的属性规则**（按类型）：
- 所有节点：`name [TYPE] WxH`
- 有 Auto Layout 的：`direction/align/gap:XX/pad:XX,sizing`
- 有背景色：`bg:#RRGGBB` 或 `bg:linear-gradient(...)`
- 有边框：`border:#RRGGBB/XXpx`
- 有圆角：`r:XX` 或 `r:XX,XX,0,0`
- 有阴影：`shadow` 或 `shadow:inset-top`
- TEXT 节点：`"文案内容" XXpx/weight/#color`
- VECTOR 节点：显示为 `[IMAGE-SVG]`
