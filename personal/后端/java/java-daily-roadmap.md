# Java 初学者每日任务版路线图  
  
## 使用方式  
一句话概括：每天按任务完成，不追求一次学很多，重点是学完立刻写代码。  
  
| 规则     | 要求                      |     |
| ------ | ----------------------- | --- |
| 每天学习时长 | 1.5 ~ 3 小时              |     |
| 每天任务结构 | 学概念 + 写代码 + 自测 + 复盘     |     |
| 完成标准   | 当天内容必须自己独立敲一遍           |     |
| 节奏原则   | 学不会就多卡一天，不要硬跳下一章        |     |
| 输出要求   | 每天至少保留 1 个 `.java` 练习文件 |     |
  
## 第一周：语法入门  
一句话概括：先学会写、编译、运行最基础的 Java 程序。  
  
| 天数    | 学习主题                 | 当日任务                                      | 完成标准                  |     |
| ----- | -------------------- | ----------------------------------------- | --------------------- | --- |
| Day 1 | 环境与第一个程序             | 安装 JDK，学会 `javac` 和 `java`，写 `HelloWorld` | 能独立编译运行 Java 文件       | ✔   |
| Day 2 | 变量与数据类型              | 练习 `int`、`double`、`char`、`boolean`，输出变量值  | 能解释基本类型区别             |     |
| Day 3 | 运算符                  | 练习算术、比较、逻辑、自增自减                           | 能写表达式并判断结果            |     |
| Day 4 | `if-else`            | 写成绩等级判断、奇偶判断、年龄判断                         | 能独立写多分支判断             |     |
| Day 5 | `switch`             | 写简单菜单程序、月份季节判断                            | 能理解 `case`、`break`、穿透 |     |
| Day 6 | `for` 循环             | 写 1~100 求和、偶数统计、九九乘法表                     | 能独立写基础循环              |     |
| Day 7 | `while` / `do-while` | 写猜数字、密码验证模拟                               | 能区分两种循环场景             |     |

## 学习中必须注意的事  
一句话概括：不要只看不写，不要只记语法不做题。  
  
| 问题 | 正确做法 |  
|---|---|  
| 看视频觉得懂了 | 必须自己关掉视频重写 |  
| 代码会看不会写 | 当天至少独立敲 2 次 |  
| 学得杂 | 按路线顺序走，不乱跳 |  
| 卡住就放弃 | 先查报错，再最小化问题，再继续 |  
| 只学语法不做项目 | 每学完一阶段必须做小项目 |  
  
## 最终目标  
一句话概括：学完后要达到“能独立写项目”，不是“知道有这个知识点”。  
  
| 阶段 | 最终能力 |  
|---|---|  
| 1 ~ 2 | 能独立写基础控制台程序 |  
| 3 ~ 4 | 能用面向对象完成小系统 |  
| 5 ~ 6 | 能处理集合、文件、异常、时间 |  
| 7 ~ 8 | 能连接数据库并写基础 Web 程序 |  
| 9 ~ 12 | 能独立完成一个 Spring Boot 后端小项目 |
